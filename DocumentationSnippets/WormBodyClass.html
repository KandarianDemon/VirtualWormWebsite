Worm Body

CyberElegans/wormbrain/MuscleSystem/WormBody

Description

Rigged armature made in blender (or any other 3D modelling and animation package) is used to generate the virtual worm body. Each bone in the armature is given a collider, a rigidbody and some CharacterJoints.

The worm body itself is only a container, which contains a skinned mesh renderer, and the body Segments, which in turn have physics components attached to them

Properties
Type Name Description
BodySegment[] segments Array of body segments
Dictionary<string,List<int>> springAnchors Holds all the anchor points for the springs, representing the muscle fibres of each muscle
Vector3 movingDirection average direction of motion
GraphContainer cont GraphContainer representing the current virtual connectome
List<Vector3> noOfMuscles number of Muscles
GameObject MuscleContainer Gameobject, that acts as parent for the Muscles on the worm
Vector3 pPoints AnchorPoints array
Methods
Type Name Description
void GetAnchorPoints() Gets all the spring anchors from each BodySegment
void GenerateMuscles() Generates Muscles in all four quadrants of the crossection of the worm. Muscle consists of 4 Muscle fibres (spring physics) and spans a total of 4 bodySegments
void SetMuscleInputs() Connect virtual muscle and the corresponding virtual motoneuron
void ScaleColliders() Adjust size of the colliders encapuslating the body segment, so that the worm tapers towards both ends
void UnpackBones() Adds BodySegment-component to all the wormbodies bones recursively.
