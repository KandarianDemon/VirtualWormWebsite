NodePhysics

CyberElegans/wormbrain/Graph/NodePhysics

Description

inerhits from UnityEngine.MonnoBehaviour

The NodePhysics-class handles the physics of the nodes, should the force directed mode be active. edges serve as springs pulling the nodes together.

This class stores the physical properties of the node, e.g rigidbodies, the nodes' mass and the collidersize. The visible body of the node and the collider are two separate objects, so that by adjusting the collider size, the distance between two nodes can be adjusted. The appearance of the nodebody is handled by the NodeVisuals-component.

Properties
Type Name Description
Rigidbody rb Rigidbody of the node
SphereCollider col SphereCollider handling the collisions of this Node
List <SpringJoint> springs Represents all the connections of this node as a List of springs. Connection strength represents the strength of the spring
List<Rigidbody rigidbodies Rigidbodies of the connected Nodes
bool _isKinematic determines wether the rigidbody of this node is set to kinematic. If true, the node wont be moved by the physics simulation, but can be animated. The node will still affect other non-kinematic objects in the simulation
float _colliderSize Sets the size of the collider
float _mass Represents the mass of the node. Highly connected nodes can be made heavier.
Methods
Type Name Description
void EnablePhysics(bool physics) toggles _isKinematic
void Init() Initializes the NodePhysics-component
void ChangeColliderSize(float size) Changes the collider-radius
void ChangeMass(float mass) Changes the rigidbody mass
void AddSpringConnection(Rigidbody connected, float strength) Adds spring between this node and a connected one
